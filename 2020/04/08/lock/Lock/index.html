方法解读：

  • acquire
  1. 先尝试获取锁，可能存在成功可能，这也是与公平的唯一区别，插入策略
  2. 加入AQS队列中

核心方法一： tryAcquire

  1. 未获取锁线程
    1. 先判断当前AQS，state值，0表示没有线程获取锁，可以通过cqs直接尝试获取锁，同时state=1，同时也代表当前线程重入次数
    2. 将AQS exclusiveOwnerThread 设置成当前线程，主要后续判断是否已持有锁的线程，重入次数+1，返回成功
    3. CQS失败
  2. 已获取锁线程
    1. 此时AQS，state>0，此时 exclusiveOwnerThread 与 当前线程一致，只需重入次数++，即可

核心方法二：acquireQueued(addWaiter(Node.EXCLUSIVE), arg))

核心方法：acquireQueued

前节点设置不为：ignal 且是初始化状态，设置为 signal

StampedLock

按位取反（~)

  • 正整数按位取反的数是它本身+1的负数。
  • 负整数按位取反的数是它本身+1的绝对值。
  • 创建锁对象，StampedLock，初始化state,state = 256
  • 获取锁

P : null

wtail : null

WHEAD: wnode

wtail : WHEAD;

p : WHEAD;

node : null

node : wnode, node.prev = WHEAD；

锁获取

  1. 第一个线程进入获取锁成功，直接返回锁版本号
  2. 第二个线程进入创建链表，whead 、wtail 形成链表，node 为当前节点，p为head 节点，当前线程堵塞
  3. 第三个线程进入，创建node节点，wtail节点为当前node节点，p为上一个节点，当前线程堵塞

锁释放

  1. 第一个线程释放锁，同时获取head节点的next节点，即第二个线程线程创建的node
  2. 此时锁获取的第二步，当前线程继续执行，此时，head节点与p为同一个指向，所以进入cqs获取锁，锁成功，此node节点成为head节点，以此类推
